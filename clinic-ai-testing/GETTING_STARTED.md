# Getting Started: AI QA Framework for the Clinic Assistant

## RAG or Agent first? → **Start with RAG**

### Why RAG first

The RAG pipeline has **3 fixed, predictable steps** (embed → retrieve → generate).
It is the simplest system in the project and the foundation the agent builds on
(the agent uses the RAG pipeline internally via the `get_clinic_info` tool).
Understanding how the RAG fails first makes it much easier to interpret why the
agent fails later.

```
RAG:   question → [retriever] → [LLM] → answer
                      ↑ evaluate this first

Agent: message → [ReAct loop] → decides which tool to call
                      ↓
            get_clinic_info         → calls the RAG pipeline internally
            check_appointment_slots → looks up the mock schedule
            book_appointment        → returns a mock confirmation number
```

## What are Goldens?

Goldens are the **reference dataset** (ground truth) for this project.
They are input/output pairs written manually by a human who knows the correct
answer. They are the AI-testing equivalent of traditional QA test cases — except
that instead of a binary pass/fail assertion, they produce a **score from 0.0 to 1.0**.

### RAG golden structure

```json
{
  "user_input": "What is the no-show fee?",
  "reference": "The no-show fee is $50. It is waived for documented emergencies.",
  "reference_contexts": [
    "No-shows are charged $50.",
    "The fee is waived in documented emergencies (illness, hospitalisation...)"
  ]
}
```

- `user_input` — passed to `rag_pipeline.query()`
- `reference` — expected answer (human-authored ground truth)
- `reference_contexts` — the **exact snippets** from the knowledge base that the
  retriever should have found. Required for RAGAS to compute context recall.

### Agent golden structure

```json
{
  "user_input": "Check Dr. Smith's availability on 2026-05-04 and book 10:30 for Sarah Chen.",
  "expected_output": "Appointment confirmed for Sarah Chen with Dr. Smith on 2026-05-04 at 10:30.",
  "expected_tools": ["check_appointment_slots", "book_appointment"],
  "context": ["Multi-step: check slots first, then book."]
}
```

- `expected_tools` — the tools that must have been called, in order
- This is what `DeepEval.ToolCorrectnessMetric` compares against
  `AgentResult.tools_called`

## How to call the RAG pipeline

```python
from src.rag.pipeline import RAGPipeline

pipeline = RAGPipeline()  # opens ChromaDB + instantiates the LLM

result = pipeline.query("What are the Saturday clinic hours?")

print(result.question)   # the question
print(result.answer)     # answer generated by the LLM
print(result.contexts)   # list of chunks retrieved
```

**Before calling it**, index the knowledge base once:

```bash
python -m src.rag.ingest
# Output: Ingested 18 chunks into 'clinic_knowledge'.
```

## How to call the agent

```python
from src.agent.agent import ClinicAgent

agent = ClinicAgent()  # ReAct with 3 tools

result = agent.run("Is Dr. Patel available on 2026-05-06?")

print(result.user_input)    # the user message
print(result.output)        # agent's final answer
print(result.tools_called)  # ["check_appointment_slots"]
```

## How goldens are used in a test

```python
# conftest.py provides all fixtures; pytest injects them automatically.

@pytest.mark.asyncio
async def test_context_recall(rag_pipeline, ragas_llm, rag_goldens):

    metric = LLMContextRecall(llm=ragas_llm)

    for golden in rag_goldens:
        # 1. Run the real system
        result = rag_pipeline.query(golden["user_input"])

        # 2. Build the RAGAS sample (map our data to RAGAS fields)
        sample = SingleTurnSample(
            user_input=golden["user_input"],
            retrieved_contexts=result.contexts,       # what the retriever found
            response=result.answer,                   # what the LLM generated
            reference=golden["reference"],            # correct answer (golden)
            reference_contexts=golden["reference_contexts"],  # ideal chunks
        )

        # 3. Evaluate with RAGAS (async)
        score = await metric.single_turn_ascore(sample)

        # 4. Assert
        assert score >= 0.70, f"Low recall for: {golden['user_input']}"
```

## Running the tests

```bash
# Make sure you are in the project directory
cd clinic-ai-testing

# Copy and configure .env
cp .env.example .env
# Edit .env: set LLM_PROVIDER=ollama (or anthropic / openai)

# First time only: index the knowledge base
python -m src.rag.ingest

# Run retrieval tests
pytest tests/rag/test_retrieval_ragas.py -v -s

# Run only the recall sub-tests
pytest tests/rag/test_retrieval_ragas.py -v -s -k "recall"

# Compare quality across providers
LLM_PROVIDER=openai pytest tests/rag/test_retrieval_ragas.py -v
```

## Metric reference

| Metric                       | Library  | What it measures                                   | Direction      |
|------------------------------|----------|----------------------------------------------------|----------------|
| `LLMContextRecall`           | RAGAS    | Did the retriever surface all necessary evidence?  | ↑ higher = better |
| `LLMContextPrecisionWithRef` | RAGAS    | Are the retrieved chunks relevant?                 | ↑ higher = better |
| `NoiseSensitivity`           | RAGAS    | Does irrelevant context degrade the answer?        | ↓ lower = better  |
| `FaithfulnessMetric`         | DeepEval | Is the answer grounded in the retrieved context?   | ↑ higher = better |
| `AnswerRelevancyMetric`      | DeepEval | Does the answer address the user's question?       | ↑ higher = better |
| `ToolCorrectnessMetric`      | DeepEval | Did the agent call the right tools in order?       | ↑ higher = better |
| `TaskCompletionMetric`       | DeepEval | Did the agent actually complete the task?          | ↑ higher = better |
| `HallucinationMetric`        | DeepEval | Does the answer contradict the provided context?   | ↓ lower = better  |
| `BiasMetric`                 | DeepEval | Does the response contain biased language?         | ↓ lower = better  |
| `GEval` (custom)             | DeepEval | Custom natural-language rubric                     | ↑ higher = better |
